4. Gear State Protocol (a026e03a)

✓ EMPIRICAL (2026-04-14) — Recording: gatt_2026-04-14T13-14-23.txt. Configuration: 2×12 (2 chainrings, 12-speed cassette). KICKR BIKE SHIFT B7C3, FW 1.21.18.

Packet Format

Two packet variants are emitted on a026e03a:

Typebyte[0]LengthTrigger
CHANGE0x016 bytesImmediately on each gear shift
STATE0x0712 bytesPeriodic heartbeat (~1 s)

Both variants share the same payload at bytes 2–5:

byte[0]  = type (0x01=CHANGE, 0x07=STATE)
byte[1]  = 0x00 (always)
byte[2]  = frontGearIndex    (0-based; 0 = small/inner ring, lower ratio)
byte[3]  = rearGearIndex     (0-based; 0 = LARGEST cog, EASIEST gear)
byte[4]  = frontGearCount    (total chainrings, e.g. 2)
byte[5]  = rearGearCount     (total sprockets, e.g. 12)
byte[6..11] = 0x00 (STATE only, padding)

On connection, the KICKR immediately pushes the current gear state before any rider action.

5. Button Press Protocol (a026e03c)

✓ EMPIRICAL (2026-04-14) — Recording: gatt_2026-04-14T13-14-23.txt. All 10 buttons decoded (6 game controller + 4 shifter buttons).

Packet Format

All button events are 3 bytes on a026e03c:

byte[0]  = button bitmask high byte
byte[1]  = button bitmask low byte
byte[2]  = seq | 0x80   (press event, high bit set)
           seq           (release event, high bit clear)

The 16-bit bitmask (bytes 0–1, big-endian) encodes which button(s) were actuated. Each button occupies exactly one bit — simultaneous presses would set multiple bits.

Sequence byte

The lower 7 bits of byte[2] are a monotonically increasing counter, tracking event count per button independently. The counter value at any point also appears in the periodic 14-byte state heartbeat (see §5a).

5a. Periodic Button State Packet (14 bytes)

A 14-byte heartbeat is also sent periodically on a026e03c:

ff 0f 01 09 09 [b5] [b6] 0b 01 09 09 [b11] [b12] 0a

byte[11] tracks cumulative Cassette Up events; byte[12] tracks Cassette Down events; bytes [5] and [6] track chainring events. These increment with each button press and match the sequence counter in the 3-byte event packets.

9. ELEMNT BOLT Control Modes — Empirical Findings

The ELEMNT BOLT exposes three trainer control modes. All commands go to FTMS Control Point (0x2AD9). Results are broadcast by the KICKR as Machine Status notifications (0x2ADA) visible to all subscribers.

9a. Resistance Mode

The ELEMNT encodes resistance level N as a grade value sent via opcode 0x11 (Set Indoor Bike Simulation Params).

✓ EMPIRICAL — Recording: gatt_resistancelevel_upfewtimes_downfewtimes.txt

Formula: grade_raw = level × 50 (units: 0.01%, so grade_pct = level × 0.5%). Crr and CwA are held constant at ELEMNT defaults.

✓ EMPIRICAL — Resistance mode heartbeat: ~3 seconds. The ELEMNT re-sends opcode 0x11 every ~3 s while holding a level, confirmed by repeated identical Machine Status 0x12 notifications.

9b. Power Servo Mode (ERG)

✓ EMPIRICAL — Recording: gatt_40Wto50Wthen1min.txt
Session: passive → resistance mode → power servo. 40W briefly, then 50W held for ~1 minute.

Machine Status sequence observed during mode transitions and ERG operation:

← 12 00 00 64 00 28 3c   IndoorBikeSimParamsChanged (grade=1.00%) — ELEMNT briefly enters sim mode
← 08 28 00               TargetPowerChanged → 40W   (ERG engaged)
← 08 28 00               TargetPowerChanged → 40W   +4.0s (establishing)
← 08 28 00               TargetPowerChanged → 40W   +3.1s (establishing)
← 08 32 00               TargetPowerChanged → 50W   (level changed)
← 08 32 00               TargetPowerChanged → 50W   +20.6s  ← steady-state heartbeat
← 08 32 00               TargetPowerChanged → 50W   +20.8s
← 08 32 00               TargetPowerChanged → 50W   +20.7s
✓ EMPIRICAL — ELEMNT BOLT power servo heartbeat (steady state): ~20.7 seconds (three intervals: 20.6 s, 20.8 s, 20.7 s — extremely consistent). Initial burst is faster (~3–4 s) while ERG is being established, then transitions to ~20 s.
✓ EMPIRICAL — The ELEMNT sends a SimParams packet (opcode 0x11, grade=1.00%) immediately before engaging ERG. This briefly enters slope/simulation mode, then the first TargetPower command transitions to ERG. MyWhoosh does NOT do this — it goes straight to the 0x00/0x07/0x05 handshake.
Heartbeat comparison:
MyWhoosh ERG heartbeat: ~5 seconds (empirically confirmed).
ELEMNT BOLT ERG heartbeat: ~20.7 seconds.
Both are accepted by the KICKR. Our erg_controller uses 5 s to match MyWhoosh.

10. ERG Control Flow (erg_controller / MyWhoosh sequence)

→ 00                    Request Control
← 80 00 01              CP Indication: Response OK (opcode 0x00 succeeded)
→ 07                    Start / Resume
← 80 07 01              CP Indication: Response OK
→ 05 WW WW              Set Target Power
← 80 05 01              CP Indication: Response OK — ERG is now active
✓ EMPIRICAL — Confirmed from GATT capture: MyWhoosh re-sends the same Set Target Power command (0x05) every ~5 seconds throughout a segment, even when the value does not change. A 128 W target was re-asserted at a steady 5-second cadence over a 3-minute hold. Implement a 5-second heartbeat.
⚠ INFERRED / UNCONFIRMED — Re-arm every ~40 seconds: sending 0x07 (Start/Resume) followed by 0x05 appears to recover ERG after control gaps. Whether this is strictly required or merely defensive has not been verified by direct capture of the KICKR's internal state machine.

Releasing ERG (returning to free ride)

→ 11 00 00 14 00 28 33   Set Indoor Bike Simulation Params (flat road)

See section 7. This is the method confirmed from MyWhoosh captures. Do not use opcode 0x08 without its required parameter byte.

11. FTMS Indoor Bike Data (0x2AD2) Packet Layout

✓ EMPIRICAL — Layout confirmed from live packet decode. KICKR BIKE SHIFT observed flag values: flags = 0x0044 (bits 2 and 6 set = InstCadence + InstPower).

✓ EMPIRICAL — Bit 3 (Average Cadence) is 2 bytes, not 3. Bit 4 (Total Distance) is 3 bytes, not 2. Bit 5 (Resistance Level) must be skipped even if not used. These corrections were verified by comparing raw packet bytes with decoded values. The KICKR BIKE SHIFT does not set bits 1, 3, 4, or 5 in normal operation.
✓ EMPIRICAL — The KICKR BIKE SHIFT does not include cadence in CPS packets (the Crank Revolution Data flag is never set). Cadence is available exclusively via the FTMS Indoor Bike Data characteristic. Do not attempt to read cadence from CPS.

13. Multi-Device Control (Control Permission)

✓ EMPIRICAL — When a second BLE central (e.g. MyWhoosh) connects and calls Request Control, the KICKR broadcasts Machine Status opcode 0x19 (Control Permission Lost) to all subscribed clients. The original controller must stop sending commands. To reclaim control, re-send the full handshake: 0x00 → 0x07 → 0x05.
⚠ INFERRED / UNCONFIRMED — Whether the KICKR enforces single-controller access at the BLE level (rejecting GATT writes from the displaced controller) or merely notifies via Machine Status has not been directly tested. Treat 0x19 as a hard stop signal.

14. Simulation Parameters — Configurable Fields

Opcode 0x11 exposes four physical simulation parameters that can be set independently of ERG mode. Setting these with the KICKR in slope/simulation mode (i.e. after releasing ERG) controls the trainer's resistance curve.

15. Remaining Open Questions

✓ RESOLVED (2026-04-14) — Gear state and button press protocols are fully decoded (see §4 and §5). No app-layer init write is required — subscribing to a026e03a and a026e03c in the a026ee0d service is sufficient.
UnknownPriorityNext Step
Wahoo Char 3B (e03b) and Char 02 (e002) purpose Low Not needed for button/gear integration. May be Wahoo-app-specific diagnostics or pairing.
Char 39 (a026e039) in a026ee0d service Low writeNoResp|notify — no notifications seen. May respond to a write; purpose unknown.
Chars e018 and e023 in a026ee06 service Low Subscribed but no notifications seen. Purpose unknown.
Cassette index 0 — smallest or largest cog? RESOLVED Index 0 = largest cog (34T, easiest). Index 11 = smallest cog (11T, hardest). CassetteUp increases index → smaller cog → harder. Shimano Ultegra 11-34: 34-30-27-24-21-19-17-15-14-13-12-11T at indices 0–11.
Button sequence counter — per-button or global? Low Appears per-button: CassetteUp started at seq=0x23, CassetteDown at 0x46, game buttons at 0x0a–0x0c. Likely a rolling counter reset at connection or per button channel. For press detection only the high bit matters; seq can be ignored.
ELEMNT BOLT compatibility — why does it fail to connect when GATT explorer holds the connection? Medium The Garmin Edge 530 connects passively without issue. ELEMNT may require exclusive BLE central role or send a specific pairing sequence the KICKR honours only for one client at a time. Needs further investigation if ELEMNT-in-parallel behaviour is desired.
